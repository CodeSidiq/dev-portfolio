========================================
DESIGN SYSTEM DOCUMENTATION
========================================

Note:
Values defined in this document represent **design reference sizes** taken directly from the Figma design.

These values act as **visual upper bounds** for responsive implementation.

Implementation rules:

• Design tokens represent the maximum visual size
• Responsive behavior must use controlled scaling
• clamp() is the preferred responsive technique
• clamp max must equal the design token
• clamp min must remain visually proportional

Example:

Design token:
DISPLAY XL = 40px

Responsive implementation example:
font-size: clamp(1.75rem, 3vw, 2.5rem);

The maximum value must always match the design token.

Random scaling is forbidden.

========================================
RESPONSIVE IMPLEMENTATION RULE
========================================

Responsive implementation must preserve visual proportions.

Preferred techniques:

• clamp()
• min()
• max()
• vw/vh when appropriate

Avoid:

• arbitrary transform scaling
• uncontrolled breakpoints
• random px overrides

Design tokens represent visual reference points.

Responsive behavior must adapt while preserving those references.

## DESIGN TOKEN RULE

All colors must use design tokens.

Example:

- bg-background
- text-foreground
- text-muted
- border-border
- bg-card

Raw hex values are forbidden inside TSX.

---

## UI PRIMITIVE RULE

Reusable UI must live in:

src/components/ui

Standard components:

- Button
- Card
- Input
- Badge
- Dialog
- Accordion

Sections must use these primitives instead of redefining styles.

---

## LAYOUT SYSTEM RULE

All sections must follow consistent layout:

- max-width container
- horizontal padding
- vertical spacing
- grid/flex alignment rules

Do not create custom layout logic per section.

---

## MOTION RULE

Animation must be:

- subtle
- purposeful
- consistent

Avoid:

- excessive motion
- random animation
- inconsistent timing

---

## ASSET RULE

Assets must follow the official project asset structure:

- public/assets/images → general images
- public/assets/icons → SVG icons
- public/assets/projects → project screenshots/images
- public/assets/avatars → profile/testimonial avatars
- public/assets/ornaments → decorative visual assets

Forbidden:

- base64 large images
- assets inside src
- inconsistent asset path
- remote placeholder services
